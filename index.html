Discord has good UI design

This website lists all the "great" design decisions made by Discord in the past and even now.
Note that the goal is not to attack anyone at Discord, but to simply show and poke fun at the mistakes they make.

Concept is inspired by the (Now sunsetted) website Twitter is going great!.

Embeds display way larger than their content¶

27/02/2023

Using the Android App, Embeds may show with a much larger height than the content itself.

Messages can have a bugged (edited) text position¶

12/02/2023

Using a specific website someone can create a bugged (edited) tag, where its position is moved to wherever you put the "(edited)" text in the site.


Bug allows posting of empty messages with embeds.¶

12/02/2023

Using a specific combination of spoilers and underlines allows you to post an empty text. Posting links that way will still render embed, making it look like you used a form of automation or API access to post embeds without message content.


Codeblock barely noticable on Mobile¶

12/02/2023

On the Mobile version of Discord are Code blocks (or inline code) barely noticable. The only thing is a thin outline for the code block and a slightly different monospaced font.

Inconsistent Emojis in Channel names¶

23/12/2022

A long standing issue in Discord is, that emojis are inconsistent when used in channel names, as Discord is using the Unicode version instead of the Twemoji version they usually use in the chats messages.

This exists since basically the start and still confuses people to this day.

Android app gets slowed down by gifs¶

17/12/2022

In a more recent update of the Android app can certain gifs cause the app to slow down (lag) significantly, while also making the phone heat up.

In addition is the a11y (accessability) setting for auto-playing gifs ignored for gifs inside embeds, resulting in gifs playing while the setting is set to "off".

Discord changes Android App from Java/Kotlin to react¶

24/08/2022

Discord has changed their Android app from being made in Java/Kotlin to be made with React.
The main reason was to remove the gap between the iOS and Android app to have a single team working on both platforms.

The result howver, was a huge collection of bugs that could be encountered without any provocation on your end. Some of them still exist to this day and new ones are encountered too.

Discord's new Brand redesign¶

13/05/2021

On the 13th of May did Discord announce their new redesign, which included changes to a lot of things, such as their Logo called "clyde" and even the Watermark.
This decision encountered quite a bit of backlash including complains about possible a11y issues with the new contrasts and colour schemes.
